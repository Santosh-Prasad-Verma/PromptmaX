How contribution works

Small changes are welcome.

You do not need to rewrite the product. Pick one useful improvement, keep it focused, and make the project better.

Start focused

One page fix, one API cleanup, one prompt template, one docs improvement, or one test case.

Explain the change

Clear PR notes matter: what changed, why it helps, and what you checked.

Respect the UI

Match the existing PromptmaX visual system, logo usage, spacing, and responsive behavior.

Leave it cleaner

Fix the actual issue without unrelated rewrites, broken routes, or dead assets.

Contribution flow

Simple path from idea to merged change.

Keep changes reviewable. The best contributions are specific, tested where useful, and aligned with the existing product feel.

Pick a task

Choose a focused bug, missing page, docs gap, prompt template, or quality check improvement.

Fork and branch

Work in a small branch, keep unrelated edits out, and follow existing file patterns.

Check the result

Verify the page, route, API, or prompt behavior you changed. Mention anything not tested.

Open a PR

Explain the problem, the fix, files touched, and how reviewers can confirm it works.

Start contributing

Start with one useful improvement.

Improve a page, write a guide, add a prompt template, fix a backend edge case, or document a workflow that helps the next user.
