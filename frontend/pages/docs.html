PromptmaX docs

Use PromptmaX to turn rough prompts into structured, scored, reusable instructions. These docs cover the local Django API, frontend playground, prompt quality checks, and saved prompt workflows.

Introduction

PromptmaX is a prompt engineering OS. It validates raw input, detects intent, builds context, applies structure, scores quality, compares versions, and returns a better instruction for AI work.

Base API path: /api/v1/

All API examples below use the Django routes exposed by this project. If you run locally with `start-server.bat`, the default host is http://127.0.0.1:8000.

Quick start

Run the server

cd backend
python manage.py migrate
python manage.py runserver 127.0.0.1:8000

Then open the playground to enhance prompts in the browser.

First enhancement request

curl -X POST http://127.0.0.1:8000/api/v1/enhance/ \
  -H "Content-Type: application/json" \
  -d "{\"prompt\":\"make a SaaS dashboard idea better\",\"enhancement_level\":\"advanced\"}"

Environment

Required local environment values should live in backend/.env. At minimum, the Django app needs a secret key.

DJANGO_SECRET_KEY=replace-with-a-long-random-secret
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost

Model provider keys are optional depending on which AI routes you use. Keep secrets out of public commits.

Enhance prompt

POST /api/v1/enhance/

Transforms a weak prompt into a structured prompt with role, task, context, constraints, output format, and checks.

ParameterTypeDescription
prompt requiredstringRaw prompt to improve.
enhancement_levelstringbasic, intermediate, advanced, or expert.
modestringenhance returns a better prompt. generate can ask the selected model to answer.
{
  "prompt": "make app idea better",
  "enhancement_level": "advanced",
  "mode": "enhance"
}

Analyze quality

POST /api/v1/analyze/

Scores prompt quality and returns intent, domain, and improvement signals.

{
  "prompt": "write landing page"
}

PromptmaX checks clarity, specificity, completeness, structure, actionability, grammar, and production constraints.

Compare prompts

POST /api/v1/compare/

Compares prompt versions so you can see which instruction is stronger before saving or shipping it.

{
  "prompt_a": "make dashboard",
  "prompt_b": "Design a SaaS dashboard with KPIs, charts, auth states, API assumptions, and QA checks"
}

Batch enhance

POST /api/v1/batch-enhance/

Processes multiple prompts in one workflow for bulk cleanup, template migration, or content operations.

{
  "prompts": [
    "write product brief",
    "make support answer better",
    "create research summary prompt"
  ],
  "enhancement_level": "intermediate"
}

URL research

POST /api/v1/analyze-url/

Queues page research and returns a task status URL for polling.

{
  "url": "https://example.com",
  "question": "Create a product positioning brief from this page"
}

Projects and assets

PromptmaX includes project, asset, and version models for reusable prompt work.

GET /api/v1/api/projects/

List prompt projects in the open workspace.

GET /api/v1/api/assets/

List saved prompt assets and their latest versions.

Prompt history

GET /api/v1/history/

Use history to inspect previous prompt runs, quality movement, and enhancement metadata.
